Description:
Este post sera bien breve ya que el binario no tomaba mucho tiempo en entender como podria devolver el flag, ahora existian dos caminos para resolver este reto uno lago y otro corto. El corto solo se necesitaba parchar el binario y obtener la flag y el largo se necesitaba analizar que hacia el binario en si. Ahora esto hace la diferencia entre un reverse y un competidor de ctf ya que el reverse aunque sabe como resolver rapidamente el reto de igual manera quiere entender el flujo y se va ael tiempo jeje.
